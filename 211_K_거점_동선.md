# K — 거점 및 동선

> 연관 파일: `210_K_프로필.md` / `212_리아프빌딩_구조.md` / `213_페어그룹_조직.md`
> 시점 기준: 2024년 현재 / 역사적 흐름 포함

---

## 거점 운용 원칙

- K의 물리적 위치는 직접보좌진 3인 외 공유되지 않는다
- 단일 거점에 장기 체류하지 않는 것이 원칙이나 리아프 빌딩 입주 이후 사실상 본부 중심 동선
- 각 거점은 독립적 법인·위장 명의로 등록
- 거점 간 이동 시 대역 동선과 실제 동선 분리 운용
- 거점 내 통신은 에어갭 또는 별도 암호화 채널 사용

## 국내 위성 거점

### 안전가옥 (리아프 빌딩 주변 위성 건물 내)

탈출 터널 경로상 확보된 위성 건물 3개소 중 일부 공간을 안전가옥 명목으로 운용한다.

| 항목 | 내용 |
|---|---|
| 위치 | 리아프 빌딩 3개 방향 / 300~500m 반경 |
| 형태 | 위성 건물 내 일부 공간 / 창고 등으로 위장 |
| 용도 | 외부 접촉 필요 시 임시 미팅 공간 / 비상 대피 중간 거점 |
| 명의 | 페어 페이퍼 컴퍼니 |

> **충돌 처리 원칙**: 기존 서사에서 "작은 건물 몇 층에서 만났다"는 설정이 있을 경우 이 안전가옥에서 진행한 것으로 처리한다.

## 비상탈출 체계

### 기본 철학

> 단일 탈출 루트는 존재하지 않는다. 3개 독립 루트가 병존하며 각각 서로 다른 위협 시나리오에 대응한다. 루트 간 연동은 있으나 하나가 차단되어도 나머지가 독립 작동한다.

---

### ROUTE 1 — 공중 탈출

#### 1-A. 공개 헬기 (옥상 헬기패드)

| 항목 | 내용 |
|---|---|
| 기체 | 의료용 헬기 1기 / 소방용 헬기 1기 |
| 등록 | 각각 의료법인·소방 지원 목적 공식 등록 / 실소유: 페어 페이퍼 컴퍼니 |
| 실제 운용 | 평시 실제 의료·소방 지원 용도 운용 / 공공기관 요청 시 출동 / 비용 리아프 빌딩 측 전액 부담 |
| 위장 효과 | 실사용 이력 축적으로 의심 원천 차단 / 공공기관 신뢰 자산으로 기능 |
| 비상 전환 | 경보 발령 시 즉시 페어 전용 운용 전환 / 공공 요청 자동 거절 처리 |
| 위치 | 50F 옥상 헬기패드 (공개 등록) |

#### 1-B. 비공개 비행수단 (50F 내부 격납)

| 항목 | 내용 |
|---|---|
| 기체 | 소형 멀티로터 eVTOL / 현실 최첨단 대비 1~2세대 앞선 수준 |
| 형태 | 2인승 / 접이식 로터 / 자율비행 + K 단독 조종 가능 |
| 격납 | 50F 내부 / 약 20평 개구부 / 외부에서 환기·채광 구조물로 인식 |
| 제원 | 항속거리 약 300~400km / 최고속도 약 350~400km/h / VTOL |
| 소음 | 현실 eVTOL 대비 50% 수준 / 도심 저공 식별 어려운 수준 |
| 스텔스 | 레이더 반사 면적 최소화 / 야간 조명 소등 가능 |
| 등록 | 국내 굴지 모빌리티 기업 시험기로 사전 등록 위장 / 해당 기업은 기체 실체 정보 없음 |
| 법적 처리 | 대관라인 암묵적 동의 / 사용 전 통보 시 사후승인 절차 / 관·경·군 전체 합의 |
| 인지 인원 | K·이실장·마이클 3인 한정 |

---

### ROUTE 2 — 지하 탈출

#### 위성 건물 체계

| 항목 | 내용 |
|---|---|
| 방향 | 3개 방향 독립 루트 |
| 경로 | 구불구불한 경로 허용 / 직선 불필요 |
| 경유 건물 | 방향별 중소규모 건물 각 1개소 / 총 3개 건물 |
| 건물 용도 | 주차장·일반 오피스·주거시설 등 평범한 건물 |
| 소유 | 전원 페어 해외 페이퍼 컴퍼니 / 건물별 별도 법인 |
| 지상부 | 원래 구조 그대로 임차·운용 / 실사용자 지하 구조 전혀 감지 불가 |
| 지하부 | 리아프 빌딩 B15와 연결 전용 탈출 터널 |
| 터널 연장 | 각 방향 300~500m |
| 터널 내부 | 차량 도로 + 측면 가이드 레일 (컨테이너 운송 가능) |
| 출구 | 각 위성 건물 지하 / 창고 위장 마감 |
| 건설 시기 | 리아프 빌딩 착공(2001.01)과 동시 착공 / 2002.12 준공 |

#### 탈출 후 연계

| 항목 | 내용 |
|---|---|
| 출구 차량 | 각 위성 건물 지하 출구 사전 배치 / 72시간 주기 교체 |
| 분기 방향 | 루트별 강남·한강·외곽 방향 분산 / 추적 분산 효과 |
| 신분 키트 | 각 출구 위장 신분 서류·현금·기기 사전 보관 |
| 단독 탈출 | K 혼자 전 구간 이탈 가능 구조 |

---

### ROUTE 3 — 직접 낙하 탈출

| 항목 | 내용 |
|---|---|
| 수단 | 고성능 점프수트 (윙수트 + 낙하산 결합형) |
| 장비 보관 | 46F 사저·47F 경호시설 즉시 착용 위치 상시 보관 |
| 발동 조건 | ROUTE 1·2 모두 차단된 최악 시나리오 |
| 낙하 구역 | 사전 지정 복수 구역 (한강변·외곽 공터 등) |
| K 능력 연동 | 초감각·신체 강화로 낙하 정밀도·착지 충격 제어 |
| 착지 후 | 이실장 팀 픽업 차량 사전 대기 / K 단독 이탈 가능 |
| 우선 발동 | 야간·저시인성 상황 우선 |

---

### 루트 발동 체계

```
위협 감지
    │
    ▼
경보 등급 판단 (이실장 or K 직접)
    │
    ├─ 1등급 (조용한 이탈)
    │     └─ ROUTE 2 — 지하 터널 차량 탈출
    │
    ├─ 2등급 (신속 이탈)
    │     ├─ ROUTE 1-A 공개 헬기 우선
    │     └─ 불가 시 ROUTE 2 병행
    │
    └─ 3등급 (건물 장악 임박 / 최악)
          ├─ ROUTE 1-B 비공개 기체 우선
          ├─ ROUTE 2 동시 발동 (이실장 팀 분산 탈출)
          └─ ROUTE 1-B도 불가 시 ROUTE 3 직접 낙하
```
