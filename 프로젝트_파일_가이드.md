# 프로젝트 파일 구조 가이드

## 목적
이 가이드는 새 채팅 세션에서 Claude가 프로젝트 파일 구조를 빠르게 파악하기 위한 참조 문서다.

---

## 파일 구조 원칙

- **번호 체계**: 그룹 단위 100번대. 중간에 파일이 추가돼도 번호 재정리 불필요.
- **정본 기준**: 연도·설정 충돌 시 각 인물 서사 파일(320~450)이 정본.
- **INDEX 파일**: 각 그룹의 인덱스 파일(310, 410, 500)이 그룹 내 파일 목록과 설정 기준을 관리.
- **K 자산 수치**: 항상 `250_K_자산_v2.xlsx` 기준.

## 확장 규칙

새 인물·그룹 추가 시:
- 기존 번호 체계에 이어서 번호 부여
- 그룹이 새로 생기면 600번대, 700번대 순으로 확장
- 각 그룹에 반드시 INDEX 파일 먼저 생성 후 세부 파일 추가
- INDEX 파일에 합류 연도, 역할, 파일명 기록

```
600_기타인물/   ← 조력자, 적대 세력 등
700_에피소드/   ← 씬 메모, 아크 설계 등
```

## 주요 설정 요약

**K 자산**: 약 5경원 (2024년 기준). 서사 내에서는 "미상 / 방대한 규모"로만 표기.

**팀 구조**:
- 직접보좌진: 이실장(현장) / 박변(법률) / 마이클(IT·데이터·허브)
- 해외운용: 찰리(미국) / 에바(유럽) / JG(중국) / 루크(중동)
- 연구소: 추후 추가

**집필 방식**: 사용자가 방향 제시 → Claude가 K 포함 전 씬 자율 집필. "1" 입력 시 다음 씬 자동 진행.

**톤**: 현실주의 소설 관습. 판타지적·과장된 요소 배제.

**K 존재감 표현**: 직접 등장 최소화. 테이블 위 메모, 짧은 답신, 팀의 움직임으로 간접 표현.
